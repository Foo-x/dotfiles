# 曖昧語辞書

要求定義書でよく見られる曖昧な表現を4カテゴリに分類し、改善例を示します。
`requirements-reviewer` スキルのサブエージェント A（個別要求品質チェック）が参照します。

---

## カテゴリ 1: 程度・量の曖昧表現

数値基準なしに「どれだけか」を表現する語。

| 曖昧語 | 曖昧な理由 | 改善例 |
|--------|-----------|--------|
| `速く` / `迅速に` / `素早く` | 許容時間が未定義 | `3 秒以内に` / `500ms 以内に` |
| `十分な` | 何が十分かが主観的 | `容量 X GB 以上の` / `精度 99% 以上の` |
| `できるだけ` | 努力目標で検証不可 | 具体的な数値目標（例: `95パーセンタイル 1 秒以内`） |
| `なるべく` | 「できるだけ」と同様 | 同上 |
| `大量の` / `多数の` | 閾値が不明 | `1000件以上の` / `同時100接続以上の` |
| `少量の` / `わずかな` | 閾値が不明 | `10件未満の` / `100KB 以下の` |
| `大きい` / `小さい` | 基準が不明 | `10MB を超える` / `1KB 未満の` |
| `長い` / `短い`（時間） | 基準が不明 | `30分以上` / `5秒未満` |
| `頻繁に` | 頻度が不明 | `1時間に1回以上` / `1分おきに` |
| `まれに` / `ほとんどない` | 頻度が不明 | `月に1回未満` / `確率 0.1% 以下` |
| `改善する` / `向上させる` | ベースラインと目標値が不明 | `現状の X から Y に向上させる` |
| `最低限` | 何が最低かが不明 | 具体的な要件の下限を定義 |

---

## カテゴリ 2: 範囲の曖昧表現

対象や条件の境界が曖昧な語。

| 曖昧語 | 曖昧な理由 | 改善例 |
|--------|-----------|--------|
| `など` / `等` | 対象の網羅性が不明 | 対象を明示的に列挙（`A、B、C`） |
| `その他` | 「その他」の範囲が不明 | 「その他」を含む場合は別要求で定義、または削除 |
| `関連する` | 何が関連するかが不明 | 関連するエンティティ・イベント・条件を明示 |
| `必要に応じて` | いつ必要かが不明 | 条件を明示（例: `エラーが発生した場合`） |
| `適宜` | 裁量が不明 | 具体的な条件とアクションを定義 |
| `通常の` / `一般的な` | 基準が不明 | 具体的な状態・値・条件を定義 |
| `一部の` | どの部分かが不明 | 対象を明示（`管理者ロールを持つユーザー`） |
| `場合によっては` | 条件が不明 | EARS の IF パターンで条件を明示 |
| `あらゆる` / `すべての` | 網羅性の検証が困難 | 対象の境界を定義し、テスト可能にする |
| `特定の` | 「特定」が何かが不明 | 具体的な属性・条件で定義 |

---

## カテゴリ 3: 主観的表現

人によって解釈が異なる感覚的・評価的な語。

| 曖昧語 | 曖昧な理由 | 改善例 |
|--------|-----------|--------|
| `適切に` | 「適切」の基準が主観的 | 何をすべきかを具体的に記述 |
| `正しく` | 「正しい」状態が未定義 | 期待する出力・状態・動作を明示 |
| `うまく` | 同上 | 同上 |
| `わかりやすく` | ユーザビリティ基準が不明 | `SUS スコア 70 以上` / `タスク完了率 90% 以上` |
| `使いやすい` | 同上 | ユーザビリティテストの合否基準を定義 |
| `安全に` | セキュリティ基準が不明 | 具体的なセキュリティ要求に分解（暗号化・認証等） |
| `堅牢に` | 品質基準が不明 | 具体的な障害対応要求に分解 |
| `効率的に` | 効率の基準が不明 | 計算量・時間・リソース使用量を定義 |
| `シンプルに` | 設計基準が不明 | 要求レベルでは削除し、設計判断へ委ねる |
| `自然に` | ユーザー感覚に依存 | ユーザーテストの定量的基準を定義 |
| `リアルタイムで` | レイテンシ上限が不明 | `100ms 以内に` / `1秒以内に反映` |

---

## カテゴリ 4: 暗黙の仮定語

文脈に依存し、前提が明示されていない語。

| 曖昧語 | 暗黙の仮定 | 改善例 |
|--------|-----------|--------|
| `ユーザーは～できる` | どのユーザーか（権限・ロール）が不明 | `認証済みの一般ユーザーは～` / `管理者ロールのユーザーは～` |
| `システムは～する` | どのコンポーネントかが不明（大規模システムの場合） | `認証サービスは～` / `APIゲートウェイは～` |
| `現在の状態を` | 「現在」が何を指すか不明 | 参照するタイミング・状態変数を明示 |
| `デフォルトで` | デフォルト値が未定義 | 具体的なデフォルト値を定義（例: `デフォルトは false`） |
| `自動的に` | いつ・何がトリガーかが不明 | WHEN/WHILE パターンでトリガーを明示 |
| `定期的に` | 周期が不明 | `24時間ごとに` / `毎月1日に` |
| `既存の～を` | 「既存」の範囲・定義が不明 | 具体的な対象を定義 |
| `元のデータ` | 「元」がどの時点かが不明 | 参照するタイミング・スナップショットを明示 |
| `必要な権限を持つ` | 「必要な権限」が未定義 | 具体的なロール・権限名を明示 |
| `正常な状態で` | 「正常」の定義が不明 | 具体的な前提条件（事前状態）を列挙 |
| `変更を保存する` | どのストレージにいつ保存するかが不明 | `DBに即座にコミット` / `セッションに一時保存` 等を明示 |
| `エラーが発生した場合` | どの種類のエラーかが不明 | エラーの種類・条件を列挙（例: `ネットワークタイムアウトが発生した場合`） |

---

## 使用上の注意

- このリストは **代表例** であり、すべての曖昧語を網羅していません
- 文脈によって曖昧でない場合もあります（例: ドメイン用語として定義済みの「大量」）
- 改善例はあくまでパターンです。実際の改善はドメイン知識と要件に基づいて行ってください
- 同じ語でも、後続に数値・条件が補完されていれば問題ありません（例: `迅速に（3秒以内）`）
